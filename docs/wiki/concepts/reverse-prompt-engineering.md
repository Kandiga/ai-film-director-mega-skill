# Reverse Prompt Engineering from Frames

Reverse prompting means extracting the likely prompt ingredients from visible frames, not claiming to know the original hidden prompt.

## Procedure

1. Extract frames from the real video.
2. Build contact sheets with frame numbers and timestamps.
3. Cluster frames visually.
4. Extract camera grammar, lighting, subject, contradiction, material texture, and transition logic.
5. Write reusable prompt blocks.
6. Add caveat: reverse-engineered from visible frames.

This supports [[pre-prophetic-cinema]] and [[ai-film-qa-loop]].

## What to extract

- Camera: macro, overhead, low, orbital, tableau, reflection.
- Light: blown window, low-key smoke, starburst, golden backlight, fog headlights.
- Motif: eye, mirror, screen, animal, sacred site, data grid.
- Composition: centered icon, negative space, grid, collage, isolated object.
- Motion implication: camera drift, smoke, fabric, water, micro-gesture.
